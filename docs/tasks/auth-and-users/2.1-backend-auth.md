# 2.1 Backend Auth

**Phase:** Auth and Users
**Depends on:** 1.2, 1.3, 1.4

## Tasks

- [ ] PB-041: Implement Argon2 password hashing and timing-safe verification
- [ ] PB-042: Implement user CRUD with role enforcement (admin can create/edit/delete, users can edit self)
- [ ] PB-043: Implement JWT token generation (user ID, role, expiry in claims, jti matches auth_sessions.id)
- [ ] PB-044: Implement JWT validation (signature, expiry, lookup in auth_sessions for revocation check)
- [ ] PB-045: Implement auth_sessions operations (create, list by user, revoke one, revoke all, revoke all except, purge expired)
- [ ] PB-046: Implement auth middleware (extract JWT from Authorization header, validate, inject user context into request)
- [ ] PB-047: Implement per-IP rate limiting for login (5/min + linear delay ramp)
- [ ] PB-048: Implement POST /api/auth/register (disabled once an admin exists)
- [ ] PB-049: Implement POST /api/auth/login
- [ ] PB-050: Implement POST /api/auth/logout
- [ ] PB-051: Implement POST /api/auth/change-password (re-verify current password, revoke all sessions, issue fresh token)
- [ ] PB-052: Implement GET /api/auth/status (returns whether any users exist)
- [ ] PB-053: Implement GET /api/auth/sessions (list caller's auth sessions)
- [ ] PB-054: Implement DELETE /api/auth/sessions/:id (revoke one)
- [ ] PB-055: Implement DELETE /api/auth/sessions (revoke all except current)
- [ ] PB-056: Wire all auth hooks (login/register/password-change/session — before/after/failed)
- [ ] PB-057: Write unit tests: Argon2 hash round-trip, timing-safe compare
- [ ] PB-058: Write unit tests: JWT generation and validation (valid, expired, revoked)
- [ ] PB-059: Write unit tests: auth middleware (valid token, missing token, expired token, wrong format)
- [ ] PB-060: Write unit tests: rate limiting (delay ramp, cap at 5s)
- [ ] PB-061: Write integration tests: full registration → login → use token → logout flow
- [ ] PB-062: Write integration tests: password change → old token revoked → new token works
- [ ] PB-063: Write integration tests: auth session list → revoke one → revoke all others

## Done When

- Users can register (first boot only), log in, and use JWT tokens
- Auth middleware rejects invalid/expired/revoked tokens with 401
- Rate limiting slows brute-force attempts
- Password changes revoke all existing sessions
- All auth hooks fire at every step
